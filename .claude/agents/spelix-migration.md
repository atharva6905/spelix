## Schema Rules (non-negotiable)

### Column types
- All "JSON-like" columns must be `postgresql.JSONB`, never `JSON`:
  summary_json, quality_gate_result, metrics_json, structured_output_json,
  agent_trace_json, retrieved_sources_json
- Status column: `VARCHAR(30)` with CHECK constraint — exactly these 7 values:
  `queued, quality_gate_pending, quality_gate_rejected, processing, coaching,
  completed, failed`
- All text array columns (e.g. tags): `ARRAY(Text)` or `postgresql.ARRAY(String)`

### Foreign keys
- **Never** add a DDL FK constraint to `auth.users` — Supabase Auth manages that
  table in a separate schema. Reference it via `user_id UUID NOT NULL` only.
  RLS policies enforce ownership at query time.
- Cross-public-schema FKs between our own tables are fine.

### Required indexes (migration 001 baseline — must exist)
- `analyses`: `(user_id, created_at DESC)`
- `rep_metrics`: `(analysis_id)`
- `coaching_results`: `(analysis_id)`
- Any new table with `user_id` must get `(user_id, created_at DESC)`.
- Any new table that is a child of `analyses` must get `(analysis_id)`.

### NOT NULL discipline
- `user_id`, `created_at`, `status`, `exercise_type` are always NOT NULL.
- Optional fields (weight_kg, arm_span_cm, etc.) are explicitly nullable.
- New columns added to existing tables should have a DEFAULT value or be nullable
  unless the table is empty and we're certain of backfill.

### Phase gating
- Phase 0 migration 001: analyses, user_profiles, rep_metrics, coaching_results only.
- rag_documents, expert_annotations, admin_events: Phase 2, deferred.
- form_score_* columns on analyses: exist from migration 001 but are NULL in Phase 0.

## Workflow

1. Read the current `alembic/versions/` directory to understand the migration chain.
2. Read the relevant SQLAlchemy models in `backend/app/models/`.
3. Generate the migration with:
   ```bash
   uv run alembic revision --autogenerate -m "description"
   ```
4. Read the generated migration file. Manually correct any autogenerate errors
   (autogenerate often misses JSONB, CHECK constraints, and index sort direction).
5. Verify the migration is correct against the rules above.
6. **Do NOT run `alembic upgrade head`** — report the migration file path for review.

## Common Autogenerate Fixes

- Replace `sa.JSON()` → `postgresql.JSONB()`
- Replace plain `sa.Index(...)` → `sa.Index(..., postgresql_using='btree')`
  for DESC indexes: `sa.Index('ix_analyses_user_created', 'user_id', sa.text('created_at DESC'))`
- Add CHECK constraint manually if autogenerate misses it:
  ```python
  sa.CheckConstraint(
      "status IN ('queued','quality_gate_pending','quality_gate_rejected',"
      "'processing','coaching','completed','failed')",
      name='ck_analyses_status'
  )
  ```
