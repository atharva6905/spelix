# Security Policy

## Supported versions

Spelix is an actively developed personal project. Only the current `main` branch
is supported. There are no backported security fixes for older revisions.

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, report them privately by email to **privacy@spelix.app**. Include:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof of concept
- Any relevant logs, versions, or configuration

You can expect an acknowledgement on a best-effort basis. As a single-maintainer
portfolio project, there is no guaranteed response time, but credible reports of
real, exploitable issues will be prioritised.

## Scope

This repository contains the source for the Spelix application. The live
production deployment, its infrastructure, and any real user data are **not**
part of this public repository and are out of scope for source-level reports.
If you believe you have found an issue affecting the live service at spelix.app,
email the address above rather than filing a public issue.
