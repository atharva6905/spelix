# Contributing to Spelix

Thanks for your interest in Spelix. A bit of context sets expectations correctly.

## What Spelix is

Spelix is a **personal portfolio project** — a science-based barbell form
coaching platform built and maintained by a single developer. It is published
as source-available under the [Business Source License 1.1](./LICENSE) primarily
so that engineers can read the code, not as an invitation to collaborative
development. There is **no team, no roadmap commitment, and no support SLA.**

## Issues

Bug reports and well-scoped observations are welcome. Please:

- Search existing issues first.
- Include reproduction steps, expected vs. actual behaviour, and environment
  details where relevant.
- Understand that issues may be closed without action at the maintainer's
  discretion — particularly feature requests outside the project's narrow scope.

## Pull requests

PRs are accepted **at the maintainer's sole discretion** and may be closed
without merge for any reason, including scope, direction, or simply available
time. Before investing effort in a non-trivial PR, open an issue to discuss it
first — unsolicited large PRs are unlikely to be merged.

By submitting a contribution, you agree that your contribution is licensed under
the same terms as the project (Business Source License 1.1, converting to Apache
License 2.0 on the Change Date stated in `LICENSE`).

## Licensing

All contributions fall under the [LICENSE](./LICENSE). If that does not work for
you, please do not submit a contribution.

## Code of Conduct

Participation is governed by the [Code of Conduct](./CODE_OF_CONDUCT.md).

## Security

Do **not** open public issues for security vulnerabilities. See
[SECURITY.md](./SECURITY.md) for private reporting.
